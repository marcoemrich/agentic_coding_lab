# research/questions-cross/

Harness-übergreifende Forschungsfragen — Vergleiche zwischen Claude Code und OpenCode.

Konvention:
- RQ-Dirs als `<chapter>-<slug>/` (z.B. `1.1-harness-effect/`).
- Frontmatter-`id:` ist die stabile Identität (z.B. `RQ-harness`).
- `factors.workflow` listet Workflow-Paare aus beiden Harnessen (z.B. `[v5-single-context-cc, v5-single-context-oc]`).
- Modell-Routing muss zwischen den Zellen identisch sein, damit der Harness-Effekt nicht mit dem Routing konfundiert ist — typisch alle Zellen auf einem Portkey-Modell (`opus-4-7-portkey`), nicht Direct-API vs Portkey gemischt.

Harness-spezifische RQs (nur Claude oder nur OpenCode) leben unter `../questions-claude/` bzw. `../questions-opencode/`.
