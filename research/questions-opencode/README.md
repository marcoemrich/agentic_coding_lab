# research/questions-opencode/

Forschungsfragen, die OpenCode-Workflows (Suffix `-oc`) als Subjekt haben.

Konvention:
- RQ-Dirs als `<chapter>-<slug>/` (z.B. `1.1-model-quality-oc/`).
- Frontmatter-`id:` ist die stabile Identität (z.B. `RQ-model-quality-oc`).
- Workflow-Werte enthalten `-oc`-Suffix (`v5-single-context-oc`, `v3-basic-tdd-oc`, …).
- Modelle: Lab-Variant-IDs für Portkey-geroutete OpenCode-Modelle (`opus-4-7-portkey`, `kimi-k2-6`, `minimax-m2-7`, `gemini-2-5-pro`, etc.).

Harness-übergreifende RQs (Claude vs OpenCode) leben unter `../questions-cross/`.
